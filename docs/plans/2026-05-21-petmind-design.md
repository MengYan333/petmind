# PetMind — 世界感知生活伴侣 PRD

## 产品概述

**产品名称**：PetMind  
**一句话定位**：一只住在桌面上的宠物，感知你的真实世界状态，通过行为和互动帮你养成习惯、获取知识和资讯。  
**目标用户**：大学生 / 年轻职场人，希望养成健康习惯但讨厌冷冰冰的 App 提醒。  
**核心差异**：所有内容通过宠物行为传递，不是聊天框；Agent 驱动决策，不是规则引擎。

---

## 核心问题

现有习惯养成 App（Forest、番茄钟、健康 App）的共同问题：
- 提醒是冷冰冰的系统通知，用户直接忽略
- 需要用户主动打开 App 才有价值
- 没有情感连接，坚持动力弱

PetMind 的解法：把提醒变成「宠物在意你」，把信息摘要变成「宠物带给你礼物」。

## 三大内容模块

### 模块一：生活类（习惯守护）

**触发逻辑**：时间触发 + 行为触发

| 场景 | 触发条件 | 宠物行为 |
|------|----------|----------|
| 喝水提醒 | 用户预设时间（如每2小时） | 宠物变干涸，眼睛凹陷 |
| 早睡提醒 | 晚上11点后仍活跃 | 宠物打哈欠，眼皮下垂 |
| 运动提醒 | 连续使用电脑60分钟 | 宠物做拉伸动作 |
| 天气感知 | 雨天 | 宠物撑小伞 |
| 天气感知 | 高温 | 宠物扇扇子冒汗 |

**用户回应（点击选择）**：
- ✅ 「好，我去做」→ 宠物开心，完成记录 +1
- ⏰ 「再等15分钟」→ 宠物撅嘴，15分钟后再次提醒
- ❌ 「今天算了」→ 宠物轻微蔫，不强迫

---

### 模块二：学习类（知识投喂）

**核心体验**：每天定时，宠物「捧着书」出现，带来用户感兴趣主题的3句话摘要。

**Agent 工作流**：
1. `get_user_topics()` — 读取用户配置的学习主题（AI、设计、编程等）
2. `fetch_latest_content(topic)` — 搜索该主题最新内容
3. `summarize_to_3_sentences(content)` — 压缩成3句话
4. `set_pet_state("learning", summary)` — 触发宠物学习状态

**用户配置**：
- 选择感兴趣的主题标签（多选）
- 设置每日推送时间
- 点击宠物展开摘要，可保存/分享

**示例输出**：
> 🐱 今天 AI 领域：Google 发布了新的多模态模型，支持实时视频理解。OpenAI 宣布 o3 模型将于下月向所有用户开放。国内百度文心大模型发布企业版定价方案。

---

### 模块三：新闻类（世界速览）

**核心体验**：宠物每天早晨「拿着报纸」出现，带来3条精选新闻摘要。

**Agent 工作流**：
1. `fetch_trending_news()` — 获取热搜/头条
2. `filter_relevant(user_prefs)` — 根据用户偏好过滤
3. `summarize_headlines(news_list)` — 每条新闻一句话
4. `set_pet_state("news", headlines)` — 触发宠物新闻状态

**用户配置**：
- 新闻偏好（科技、社会、国际、体育）
- 推送时间（早上8点）

## Agent 核心决策（面试亮点）

### 多信号优先级判断

当多个触发条件同时满足时，Agent 调用 `prioritize_actions()` 做决策：

**示例场景**：
- 现在是晚上11:30
- 用户已连续使用电脑2.5小时
- 还没喝水（距上次喝水4小时）
- 有未读学习摘要

**模板逻辑（做不到）**：三个提醒同时弹出，用户烦躁关掉。

**Agent 决策（真正的价值）**：
> 当前最优先：早睡健康 > 喝水 > 学习可以明天。  
> 调用 `set_pet_state("sleepy")`，宠物催睡觉。  
> 将喝水提醒合并进去：「去喝杯水然后睡觉吧」。  
> 学习摘要推迟到明天早上。

这个组合判断，模板无法穷举，LLM 推理才能处理。

### 工具调用列表

```javascript
// 传感器工具
get_weather(location)           // 获取天气
get_screen_active_time()        // 获取当前连续活跃时长
get_current_time()              // 获取时间

// 内容工具  
fetch_news(categories)          // 获取新闻
fetch_learning_content(topics)  // 获取学习内容
summarize_content(text, style)  // 内容摘要

// 决策工具
prioritize_actions(signals)     // 多信号优先级判断
set_pet_state(state, payload)   // 设置宠物状态和内容

// 习惯工具
log_habit_completion(habit_id)  // 记录习惯完成
get_habit_history(days)         // 获取习惯历史
schedule_next_reminder(habit)   // 安排下次提醒
```

## 宠物养成系统

**成长阶段**（由习惯完成率决定）：
- 🥚 蛋 — 刚开始
- 🐣 幼崽 — 完成率 > 30%（7日均值）
- 🐱 成长期 — 完成率 > 60%
- 😺 成年期 — 完成率 > 85%

**养成互动**（随时触发）：
- 喂食 → 饱食度 +10，影响宠物活跃程度
- 梳毛 → 好感度 +5，解锁新表情
- 玩耍 → 心情 +15，解锁新动作

**解锁系统**：
- 连续7天完成习惯 → 解锁新装扮
- 阅读30条学习摘要 → 解锁「学者帽」
- 首次雨天互动 → 解锁彩虹伞

## Agent 推理可视化（核心技术亮点）

**这是与知识库类产品的关键差异**：不只是调用 AI，而是把 agent 的决策过程展示出来。

### UI 设计：思考抽屉

每次 agent 触发时，界面右下角出现可展开的「思考面板」：

```
🤔 Agent 正在分析...
  ├─ ✅ get_current_time()        → 23:32
  ├─ ✅ get_screen_active_time()  → 147分钟
  ├─ ✅ get_weather()             → 晴，22°C
  ├─ ✅ get_habit_history(7)      → 今日喝水：0次
  └─ 🧠 prioritize_actions()     → 推理中...

💡 决策结果：
   睡眠健康 > 补水 > 学习推迟
   → 触发「催眠+喝水」合并提醒
```

**为什么这很重要**：
- 面试官能实时看到 agent 调用了哪些工具
- 能看到推理链，不是黑盒
- 清晰展示「这不是 if-else，是真正的 agent 决策」

## 面试讲解要点

**为什么用 Agent 而不是规则引擎？**
> 「信号的组合是无穷的。晚上11点+连续工作3小时+雨天+未喝水，这四个条件同时存在时，应该提醒什么、怎么提醒、什么推迟——if-else 写不完。用 Claude function calling，agent 自己决定调用哪些工具、按什么顺序、输出什么结果。」

**为什么选宠物作为载体？**
> 「提醒本身不难，难的是用户愿不愿意响应。宠物制造了情感连接，让'被提醒'变成'宠物在意我'，完成率和持续使用率会更高。」
