# PetMind 产品方案：环境感知型桌面生活助手

更新时间：2026-05-26

## 1. 产品定位

PetMind 是一个**持续运行在桌面上的 AI 生活助手**，帮助用户把脑子里散落的灵感、习惯、计划变成每天可执行的行动。

核心问题：人有很多想法，但很难把模糊的灵感变成具体的行动步骤，更难坚持执行。

核心答案：用一个始终在你桌面上的 AI Agent，帮你记录、拆解、规划、提醒、回顾。

## 2. 核心价值

### 2.1 对用户的价值

| 痛点 | PetMind 的解决方式 |
|---|---|
| 灵感来了记不住，过后就忘 | 随时自然语言输入，AI 结构化存储 |
| 记下来了但不知道怎么开始 | AI 把模糊灵感拆解成具体步骤 |
| 不知道今天该做什么 | 每天早上自动生成推荐日程 |
| 传统提醒太死板，时间到了不一定适合 | Agent 综合判断当前状态，在合适时机提醒 |
| AI 给的计划不一定符合我的想法 | 通过对话协商调整优先级和安排 |
| 做了很多但没有回顾 | 每天晚上自动总结完成情况和模式 |

### 2.2 与传统 Todo/提醒应用的区别

传统应用：用户手动输入 → 手动设置时间 → 到点提醒 → 完成

PetMind：自然语言输入 → AI 理解意图并拆解 → Agent 自主判断最佳提醒时机 → 对话协商调整 → 智能回顾

**关键差异：用户只需要"说"，AI 来"想"和"推"。**

## 4. 模块详细设计

### 4.1 自然语言输入

**目标**：用户用一句话表达任何想法，AI 理解并结构化存储。

**输入示例**：

| 用户输入 | AI 理解 |
|---|---|
| "每天提醒我学多邻国" | kind: habit, cadence: 每天 |
| "每周六洗衣服" | kind: plan, cadence: 每周六 |
| "我想探索垂类 Agent" | kind: idea, 需要拆解 |
| "今天下午想读一篇论文" | kind: plan, 一次性 |
| "最近总觉得压力大" | kind: idea, 情绪类，触发陪伴 |
| "帮我找个好用的笔记软件" | kind: idea, 有明确目标 |

**AI 解析输出**：

```json
{
  "title": "探索垂类 Agent",
  "kind": "idea",
  "rawText": "我想探索垂类 Agent",
  "steps": [
    "调研垂类 Agent 的代表产品",
    "选一个感兴趣的垂类深入了解",
    "用 Claude API 搭一个最小原型"
  ],
  "cadenceLabel": "按需推进",
  "priority": "medium",
  "tags": ["学习", "AI", "探索"]
}
```

**与当前实现的差距**：当前 `parsePlanIntent` 只做简单分类（habit/plan/idea），没有拆解步骤的能力。

### 4.2 任务池（Task Pool）

**目标**：所有灵感、习惯、计划的统一存储，带完整状态管理。

**任务数据结构**：

```json
{
  "id": "task_1716700000",
  "title": "探索垂类 Agent",
  "rawText": "我想探索垂类 Agent",
  "kind": "idea",
  "status": "pending",
  "priority": "medium",
  "tags": ["学习", "AI"],
  "steps": [
    { "id": 1, "text": "调研代表产品", "done": false },
    { "id": 2, "text": "选一个垂类深入了解", "done": false },
    { "id": 3, "text": "搭最小原型", "done": false }
  ],
  "cadence": null,
  "createdAt": 1716700000000,
  "updatedAt": 1716700000000,
  "completedAt": null,
  "snoozedUntil": null,
  "notes": []
}
```

**任务状态机**：

```
        ┌──────────┐
   ──→  │ pending  │ ←──┐
        └────┬─────┘    │
             │          │
             ▼          │
        ┌──────────┐    │
        │ active   │    │  snooze
        └────┬─────┘    │
             │          │
      ┌──────┼──────┐   │
      ▼      ▼      ▼   │
 ┌────────┐ ┌────────┐ ┌────────┐
 │completed│ │ missed │ │snoozed │──→
 └────────┘ └────────┘ └────────┘
```

- **pending**：刚录入，还没开始
- **active**：今天被推荐 / 用户选择开始
- **completed**：标记完成
- **missed**：推荐了但当天没完成
- **snoozed**：用户说"先不做这个"

**与当前实现的差距**：当前 `useHabits` 只有 add/remove，没有状态、没有步骤、没有时间管理。

### 4.3 每日计划生成

**目标**：每天早上根据任务池 + 用户习惯，生成一份"今天推荐做什么"。

**触发时机**：
- 用户打开应用时，检测是否已生成今日计划
- 用户主动请求"帮我规划今天"

**生成逻辑**：

```
输入：
  - 任务池中所有 pending/active 任务
  - 今天是周几（周期任务匹配）
  - 用户近期完成模式（偏好分析）
  - 当前时间

输出：
  - 今日推荐任务列表（按优先级排序）
  - 每个任务的建议时间段
  - 可选的"探索时间"（从 idea 类任务中挑一个推荐）
```

**输出示例**：

```
今日推荐 (5月26日 周一)

必做：
  1. 学多邻国 (习惯 · 建议晚间)
  2. 周六洗衣服 (已完成 ✓)

推荐探索：
  3. 花 30 分钟调研垂类 Agent 的代表产品
     → 这是你上周提到的灵感，可以先从搜索开始

弹性：
  4. 读一篇论文 (今天下午有空的话)
```

**与当前实现的差距**：完全没有这个模块。

### 4.4 对话协商

**目标**：用户通过对话调整 AI 生成的计划，而不是只能接受或拒绝。

**对话场景**：

| 用户说 | AI 做什么 |
|---|---|
| "今天不想做垂类 Agent" | 标记任务 snoozed，调整今日计划 |
| "把学多邻国排到第一个" | 调整今日计划优先级 |
| "垂类 Agent 那个，第一步具体怎么做？" | 展开步骤详情，给出具体建议 |
| "我觉得压力好大" | 切换到情绪陪伴模式 |
| "帮我查一下有哪些垂类 Agent" | 调用搜索工具，返回结果 |
| "今天做了很多，帮我总结一下" | 生成当日总结 |

**对话不是闲聊，是计划的控制面板。**

**与当前实现的差距**：当前聊天只是"可爱猫咪人格回复"，没有和任务池打通。

### 4.5 主动弹窗提醒

**目标**：Agent 在合适的时机主动弹出提醒，而不是死板的定时触发。

**触发逻辑**：

```
Agent 决策循环（每 10 分钟）：
  1. 检查今日计划中未完成的任务
  2. 检查当前环境状态（屏幕活跃时长、是否空闲、当前时间）
  3. 综合判断是否现在是合适的提醒时机
  4. 如果合适 → 弹窗提醒
  5. 如果不合适 → 等下次循环
```

**判断"合适时机"的信号**：

| 信号 | 含义 |
|---|---|
| 用户刚从忙碌变为空闲 | 可能适合切换任务 |
| 屏幕活跃超过 2 小时未休息 | 该提醒休息了 |
| 今天有个 idea 任务还没动 | 推荐花一点时间探索 |
| 接近某个计划的时间窗口 | 提醒该准备了 |
| 用户长时间空闲 | 可以推荐探索类任务 |

**弹窗示例**：

```
┌─────────────────────────────┐
│ 🐱 你已经连续工作 2 小时了    │
│                             │
│ 要不要花 20 分钟探索一下      │
│ "垂类 Agent"？换换脑子～     │
│                             │
│  [好的]  [等会儿]  [不做这个] │
└─────────────────────────────┘
```

**与当前实现的差距**：当前弹窗只做久坐/喝水提醒，没有和任务池关联。

### 4.6 每日总结

**目标**：晚上自动生成当日回顾，帮助用户看到自己的进展。

**总结内容**：

```
今日回顾 (5月26日)

完成：
  ✓ 学多邻国 (连续第 3 天)
  ✓ 调研垂类 Agent 代表产品 (探索任务第一步)

未完成：
  ○ 读一篇论文 (可以明天再安排)

发现：
  - 你最近一周有 3 天在探索 AI 相关的内容
  - 学多邻国的坚持很好，继续保持
  - 建议明天继续垂类 Agent 的第二步：选一个深入了解
```

**与当前实现的差距**：当前 ReflectionSkill 只做简单事件计数，没有任务完成分析和模式识别。

## 5. 体现的 Agent 能力

这个产品不是"一个带 AI 的 Todo 应用"，而是一个**完整的 Agent 系统**。以下每个模块都对应 Agent 的核心能力：

### 5.1 意图理解（Intent Understanding）

**体现位置**：自然语言输入 → 结构化任务

**能力展示**：
- 把"我想探索垂类 Agent"识别为 `idea` 类型，而不是 `habit`
- 把"最近压力大"识别为情绪类输入，触发陪伴而非任务创建
- 把"每天学多邻国"理解为周期性习惯，自动设置 cadence

**技术要点**：few-shot prompting + structured output + fallback heuristic

### 5.2 多步推理（Multi-step Reasoning）

**体现位置**：灵感拆解

**能力展示**：
- 用户只说了一句话，AI 推理出 3-5 个可执行步骤
- 步骤之间有逻辑顺序（先调研 → 再选型 → 再动手）
- 步骤粒度合适（不会太粗也不会太细）

**技术要点**：chain-of-thought prompting，让模型先分析再拆解

### 5.3 状态推断（State Inference）

**体现位置**：主动弹窗的触发判断

**能力展示**：
- 不是"时间到了就提醒"，而是综合多个信号判断
- 识别用户当前是忙碌、空闲、疲劳还是适合探索
- 根据状态选择不同的提醒策略和内容

**技术要点**：规则引擎 + 多信号加权，当前 `StateInference.js` 已有基础

### 5.4 优先级决策（Priority Decision）

**体现位置**：每日计划生成

**能力展示**：
- 从任务池中选出今天最值得做的几件事
- 考虑任务类型（习惯 vs 计划 vs 灵感）、时间因素、用户偏好
- 给出排序理由，而不是黑盒输出

**技术要点**：评分函数 = 任务紧急度 × 类型权重 × 用户偏好 × 时间匹配度

### 5.5 工具调用（Tool Calling）

**体现位置**：对话中用户请求具体帮助

**能力展示**：
- 用户说"帮我查一下垂类 Agent"→ 调用搜索工具
- 用户说"帮我总结今天做了什么"→ 调用总结工具
- 用户说"第一步具体怎么做"→ 调用推理工具给出详细建议

**可调用的工具**：

| 工具 | 用途 |
|---|---|
| `search` | 搜索背景资料 |
| `decompose` | 把大目标拆成子步骤 |
| `summarize` | 总结完成情况 |
| `analyze` | 分析行为模式 |
| `suggest` | 给出下一步建议 |

**技术要点**：function calling / tool use API

### 5.6 自主行动（Autonomous Action）

**体现位置**：主动弹窗 + 每日计划自动生成

**能力展示**：
- 用户没有主动请求，Agent 自己判断"现在应该提醒用户了"
- 每天早上自动生成计划，不需要用户触发
- 晚上自动生成总结

**与被动工具的区别**：传统 Todo 应用等用户操作，PetMind 主动推动用户行动。

### 5.7 记忆与学习（Memory & Learning）

**体现位置**：任务池积累 + 每日总结中的模式发现

**能力展示**：
- 记住用户输入过的所有灵感和计划
- 发现用户的行为模式（"你最近一周有 3 天在探索 AI"）
- 根据历史完成情况调整推荐策略

**技术要点**：ActivityLog + Memory + 每日 digest 累积

### 5.8 人机协作（Human-AI Collaboration）

**体现位置**：对话协商

**能力展示**：
- AI 给出建议，用户可以调整
- 不是"AI 说了算"，也不是"用户全手动"
- 通过对话达成共识

**与纯 AI 生成的区别**：AI 是助手不是决策者，用户保持控制权。

## 7. 实施路线

### Phase 1：任务池 + 自然语言输入（基础）

**目标**：把当前的 `useHabits` 升级为完整的任务池

- [ ] 重新设计任务数据结构（增加 status, steps, priority, tags）
- [ ] 扩展 `parsePlanIntent` 支持灵感拆解（增加 steps 输出）
- [ ] 实现任务状态机（pending → active → completed/missed/snoozed）
- [ ] 改造 HabitPanel 为 TaskInput + TaskPool 列表
- [ ] 支持任务的修改、删除、归档

**产出**：用户可以自然语言输入任何想法，AI 结构化存储到任务池。

### Phase 2：每日计划 + 对话协商

**目标**：每天自动生成推荐计划，用户可以通过对话调整

- [ ] 实现 PlanGenerator Skill（从任务池生成每日计划）
- [ ] 实现 DailyPlan 展示组件
- [ ] 改造聊天为"计划协商"模式（对话与任务池打通）
- [ ] 支持对话中的任务操作（调整优先级、snooze、查看详情）

**产出**：每天有推荐计划，可以通过对话调整。

### Phase 3：主动提醒 + 每日总结

**目标**：Agent 主动在合适时机提醒，晚上自动总结

- [ ] 实现 TaskReminder Skill（替代 WellnessSkill，关联任务池）
- [ ] 扩展 StateInference 增加任务相关的状态判断
- [ ] 实现 DailyReflection Skill（替代 ReflectionSkill，分析任务完成情况）
- [ ] 弹窗提醒与任务池关联（"你还有个灵感没探索"）

**产出**：Agent 主动推动用户行动，每天有回顾。

### Phase 4：Agent 能力增强

**目标**：展示更强的 Agent 能力

- [ ] 实现 Tool Calling（搜索、拆解、分析）
- [ ] 增加行为模式识别（"你最近总在探索 AI"）
- [ ] 优化优先级决策算法
- [ ] 增加环境信号（窗口识别、学习场景）

**产出**：完整的 Agent 闭环，有工具调用和模式发现能力。

## 9. 总结

PetMind 从"桌面宠物 + 简单提醒"升级为"桌面生活助手"，核心转变是：

1. **从被动到主动**：不只是等用户操作，Agent 主动规划和提醒
2. **从死板到智能**：不是固定时间提醒，而是综合判断最佳时机
3. **从碎片到系统**：灵感 → 拆解 → 规划 → 执行 → 回顾，完整闭环
4. **从工具到伙伴**：通过对话协商，AI 是助手不是命令执行器

这个产品同时展示了 Agent 的七项核心能力：意图理解、多步推理、状态推断、优先级决策、工具调用、自主行动、人机协作。
